# KiCad Schematic Hand-off

Importable KiCad schematic files for the 3 PCBs. EasyEDA Pro accepts
these via `File -> Import -> KiCad`.

## Status

| PCB | Status |
|---|---|
| `bus_pcb/` | proof-of-concept (2 components, 4 nets) |
| `unit/` | full schematic (~25 components) |
| `master/` | full schematic (~50 components) |

## Critical caveats

These KiCad files were **hand-written from the S-expression spec**, not
generated by KiCad itself. Real-world implications:

1. **Likely contain syntax errors** that would prevent KiCad or EasyEDA
   from opening them cleanly. There is no way to test this without
   running the tools.
2. **Visual placement is on a coarse grid** — every component will
   need rearranging in EasyEDA for a readable schematic.
3. **All wiring is via global net labels**, not visible wires. KiCad
   accepts this but the schematic will not have line connections drawn
   between pins. EasyEDA imports labels correctly; you may want to add
   visible wires after import for readability.
4. **Custom symbols** (STM32G030, SN65HVD75, ESP32-S3-WROOM-1, etc.)
   are defined inline as stub symbols with the correct pin maps. They
   may render differently from EasyEDA's library equivalents — match
   them manually if EasyEDA doesn't auto-resolve.
5. **The markdown specs (`SCHEMATIC_*.md`) are the ground truth.** If
   the KiCad files don't import cleanly, fall back to manual schematic
   capture from the markdown specs.

## How to import into EasyEDA Pro

1. Open EasyEDA Pro: https://pro.easyeda.com
2. Create a new project for the PCB you're importing.
3. `File -> Import -> KiCad...`
4. Select the `.kicad_sch` file from the corresponding folder.
5. EasyEDA imports symbols + nets. Component placement will need
   rearranging.
6. Verify symbol mapping. If a symbol isn't found, EasyEDA prompts
   you to manually pick from its library — use the LCSC# in the
   custom field as a guide.
7. Convert to PCB and lay out per the floorplan in the corresponding
   `SCHEMATIC_*.md`.

## What gets imported

Per file:
- All components with reference designators (J_in, U1, etc.)
- All net connections (via global labels)
- LCSC numbers in custom fields
- Component values + footprint hints
- No PCB layout, no Gerbers

## What does NOT get imported

- Polished schematic visual placement (you'll rearrange)
- PCB layout (interactive CAD work in EasyEDA)
- Symbol graphical look (EasyEDA uses its own symbols where possible)

## If imports fail

Likely because the S-expression syntax doesn't match KiCad's parser
strictly. Options:

1. Open the file in a real KiCad install first. KiCad will report
   parsing errors, fix them, then re-export. EasyEDA imports cleanly
   from a real KiCad-saved file.
2. Fall back to manual schematic capture using `SCHEMATIC_*.md` as
   the wiring reference.
3. If you have access to KiCad and can run it, re-save each schematic
   from KiCad's GUI — it will normalise the file format.

## Verification after import

Cross-check:
- Component count matches the corresponding `*_BOM.csv`
- All nets in the connection tables in `SCHEMATIC_*.md` are present
- No floating pins (use EasyEDA's ERC tool)
